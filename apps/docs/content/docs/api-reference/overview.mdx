The Quiz App provides a comprehensive REST API for programmatic access to all quiz functionality.

## Base URL

```
https://yourapp.com/api
```

For local development:

```
http://localhost:3000/api
```

## Authentication

All API endpoints require authentication via API key. Include your API key in the `x-api-key` header:

```bash
curl -H "x-api-key: your_api_key_here" \
  https://yourapp.com/api/quizzes
```

See [API Authentication](/api-reference/authentication) for details.

## Rate Limiting

All API requests are rate-limited to **100 requests per minute** per API key.

Rate limit headers are included in every response:

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1640000000
```

When rate-limited, you'll receive a `429 Too Many Requests` response:

```json
{
  "error": "Rate limit exceeded. Please try again later."
}
```

## Pagination

List endpoints support pagination via query parameters:

- `page` ‚Äî Page number (1-based, default: 1)
- `limit` ‚Äî Items per page (default: 30, max: 100)

Example:

```bash
curl -H "x-api-key: your_key" \
  "https://yourapp.com/api/quizzes?page=2&limit=20"
```

Response includes pagination metadata:

```json
{
  "quizzes": [...],
  "pagination": {
    "page": 2,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

## Content Type

All requests and responses use `application/json`:

```http
Content-Type: application/json
```

Include this header when sending data:

```bash
curl -X POST \
  -H "x-api-key: your_key" \
  -H "Content-Type: application/json" \
  -d '{"title":"My Quiz"}' \
  https://yourapp.com/api/quizzes
```

## CORS

The API supports Cross-Origin Resource Sharing (CORS) for client-side applications. All origins are allowed in development mode. In production, configure allowed origins via environment variables.

## Versioning

The API is currently **unversioned**. Breaking changes will be communicated with a migration guide and deprecation period.
